# 我的收藏功能迭代 PRD

- 文档版本：v1.0
- 创建日期：2026-06-26
- 状态：待评审
- 关联仓库：`shougang-group-knowledge-portal`（portal 前后端）、`bisheng_2`（收藏数据与知识库底层）

## 1. 背景与目标

当前收藏功能的交互是「重动作」：用户点击收藏按钮后，弹出 `FavoriteDocumentModal`，需手动选择一个目标个人知识库，再将文档**复制**进去。该方式存在三个问题：

1. 操作步骤多，不符合「收藏」应有的轻量心智。
2. 复制副本无法反映源文件状态（源文件删除后副本仍在，信息陈旧）。
3. 收藏按钮无「已收藏 / 未收藏」状态区分，用户无法判断某文件是否已收藏，易重复收藏。

本次迭代将收藏重构为「一键 toggle + 引用模型 + 固定收藏库」，目标：

- 收藏变为一键操作，落入固定的「我的收藏」知识库。
- 收藏存引用而非副本，源文件删除后能反映为「已失效」。
- 所有展示收藏按钮的位置具备「已收藏 / 未收藏」两态切换。
- 未登录不可收藏。

## 2. 名词定义

| 名词 | 含义 |
| --- | --- |
| 我的收藏（收藏库） | 每个登录用户内置的、系统默认存在、不可删除的特殊个人知识库 |
| 收藏项 | 收藏库中的一条记录，本质是指向源文件的引用 |
| 源文件 / 源知识库 | 被收藏文件的原始位置（源知识库 ID + 源文件 ID） |
| 已失效 | 源文件已被删除，导致该收藏项不可用的状态 |

## 3. 需求范围

对应用户提出的 5 条需求：

1. 个人知识库中新增「我的收藏」知识库，默认存在、不可删除。
2. 「我的收藏」仅能查看收藏内容或取消收藏，不能进行任何其它操作。
3. 源文件被删除后，对应收藏项状态变为「已失效」。
4. 所有展示收藏按钮的位置具备「已收藏 / 未收藏」状态切换。
5. 未登录不能收藏。

## 4. 功能详细设计

### 4.1 「我的收藏」知识库

- **归属与展示**：出现在「个人知识库」列表中，每个登录用户一个，名称固定为「我的收藏」。
- **创建时机**：懒创建。用户首次发生收藏动作（或首次进入个人知识库列表）时，由后端确保该库存在；不存在则自动创建。
- **类型标记**：以特殊标记（如 `is_favorite_space=true`）与普通个人知识库区分，供前端识别并施加只读限制、供后端拒绝删除/重命名。
- **不可删除 / 不可重命名**：
  - 前端：在知识库列表与详情页隐藏/禁用「删除」「重命名」入口。
  - 后端：对该库的删除、重命名请求一律拒绝并返回明确错误码，保证即便绕过前端也无法破坏。

### 4.2 收藏 = 一键 toggle（引用模型）

- **交互**：收藏按钮为两态切换。
  - 未收藏 → 点击 → 加入「我的收藏」→ 按钮变为已收藏态。
  - 已收藏 → 点击 → 取消收藏 → 按钮变为未收藏态。
- **去除选库弹窗**：原 `FavoriteDocumentModal`「选择目标知识库」流程不再用于主收藏流程。
- **存储为引用**：收藏记录字段至少包含：
  - `user_id`（隐式由登录态确定）
  - `source_space_id`（源知识库 ID）
  - `source_file_id`（源文件 ID）
  - `created_at`（收藏时间）
  - 唯一约束：同一用户 + 同一源文件只能有一条收藏记录（防重复收藏）。
- **查看**：进入「我的收藏」点击有效收藏项时，定位并打开/预览其**源文件**（读取源知识库中的原文件），而非副本。

### 4.3 收藏项状态：有效 / 已失效

- **有效**：源文件存在，可正常打开/预览。
- **已失效**：源文件已被删除。
  - 触发与判定：在「我的收藏」列表加载时，对每个收藏项实时校验源文件是否仍存在；不存在则判定为已失效。（采用读取时判定，避免依赖源文件删除事件的实时回调。）
  - 失效项表现：置灰展示并打上「已失效」标签；不可打开 / 不可预览。
  - 失效项允许的操作：**仅「取消收藏」**，由用户自行清理。

### 4.4 「我的收藏」只读受限

进入「我的收藏」知识库后，**仅允许**：

- 查看收藏列表。
- 打开 / 预览有效收藏项的源文件。
- 取消收藏（有效项与已失效项均可取消）。

**禁用**以下全部操作（前端隐藏入口 + 后端兜底拒绝）：

- 上传 / 新增文件
- 删除文件（删除文件即「取消收藏」，不提供独立删除）
- 删除 / 重命名该知识库
- 移动 / 复制文件到其它库
- 标签编辑
- 关联为新版本
- 知识库问答等其它一切操作

### 4.5 收藏按钮全局状态一致

- **作用范围**：所有展示收藏按钮的位置统一遵循同一两态渲染逻辑，包括但不限于：
  - 搜索结果列表（SearchPage）
  - 知识库内文件列表（ListPage）
  - 未来扩展位置（文档预览页、文档详情页、相关文档列表等）
- **状态来源**：列表加载时**批量查询**当前用户对这批文件的收藏状态（一次请求返回 `source_file_id → 是否已收藏` 的映射），避免逐条请求。
- **toggle 后即时反馈**：收藏 / 取消后立即更新该按钮状态；同一页面中指向同一源文件的多个按钮应保持一致。

### 4.6 登录约束（未登录不可收藏）

- **按钮可见性**：未登录用户在所有位置**直接隐藏**收藏按钮。
- **接口层**：收藏 / 取消收藏 / 批量查询收藏状态 / 「我的收藏」列表等接口均要求登录态，未登录返回未授权错误。

## 5. 接口设计（建议）

> 具体路径与字段在实现阶段对齐现有 `knowledge` 路由风格，以下为功能契约。

| 能力 | 方法 | 说明 |
| --- | --- | --- |
| 收藏（加入我的收藏） | `POST /api/v1/knowledge/favorites` | 入参 `{ source_space_id, source_file_id }`；自动确保收藏库存在；幂等（已收藏则返回已存在） |
| 取消收藏 | `DELETE /api/v1/knowledge/favorites` | 入参定位 `{ source_space_id, source_file_id }` 或收藏项 ID |
| 批量查询收藏状态 | `POST /api/v1/knowledge/favorites/status` | 入参文件列表，返回各文件是否已收藏 |
| 我的收藏列表 | `GET /api/v1/knowledge/favorites` | 返回收藏项列表，每项含源文件信息与 `status: valid \| invalid` |

- 原 `POST /favorites`（带 `target_space_id` 的复制语义）调整为引用语义；如需兼容历史，评估迁移方案（见第 7 节）。

## 6. 边界与异常

- **重复收藏**：唯一约束保证幂等；接口对已收藏返回成功/已存在，不报错打断体验。
- **源文件在收藏后被删**：下次进入收藏列表时显示「已失效」；不主动推送。
- **源知识库整体被删**：其下收藏项同样判定为已失效。
- **未登录点击（理论上按钮已隐藏）**：接口层兜底返回未授权。
- **多端 / 多标签状态不同步**：以接口返回为准，页面刷新或重新加载列表时校正。

## 7. 数据迁移与兼容

- 历史「复制副本」式收藏数据与新「引用」式收藏并非同一模型。需在实现阶段确认：
  - 是否需要将历史副本迁移为引用记录；
  - 历史复制进各个人知识库的副本如何处理（保留 / 清理 / 提示用户）。
- 该项作为实现前置确认项，不在本 PRD 强制结论。

## 8. 验收标准

1. 个人知识库列表中每个登录用户均可见「我的收藏」，且无删除/重命名入口；后端拒绝对其删除/重命名。
2. 任一收藏按钮位置：未收藏点击后变已收藏并出现在「我的收藏」；再次点击取消并从「我的收藏」移除。
3. 收藏后删除源文件，再进入「我的收藏」，该项显示「已失效」、不可打开、仅可取消收藏。
4. 搜索页与列表页对同一已收藏文件均显示已收藏态；刷新后保持一致。
5. 未登录状态下所有位置均不显示收藏按钮；直接调用收藏接口返回未授权。
6. 在「我的收藏」中无法执行上传、删除文件、移动、标签编辑、关联新版本、问答等操作。

## 9. 不在本次范围（YAGNI）

- 收藏分组 / 收藏夹多级目录。
- 收藏排序 / 备注 / 标签。
- 收藏内容的全文检索与问答。
- 收藏数量上限与配额。
- 收藏的分享 / 协作。
