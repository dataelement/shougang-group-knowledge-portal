# PRD — 热门问题展示与搜索页筛选改造

> 状态：待评审 · 创建日期：2026-06-03 · 适用仓库：shougang-group-knowledge-portal（门户）
> 关联：BiSheng 侧无改动；依赖门户已落地的「检索范围按登录态切换」规则（见 §7）。

---

## 1. 背景与现状

- 首页搜索框下方的「热门搜索」当前展示的是**热门标签**（`/knowledge/home` 返回的 `tags`，前端 `rankedHotTags`，见 [HomePage.tsx:490](../frontend/src/pages/HomePage.tsx#L490)、[L600-614](../frontend/src/pages/HomePage.tsx#L600-L614)），点击后跳到按**标签**过滤的搜索页（`buildTagSearchPath`）。
- 「热门问题」其实已有数据底座：`qa.hot_questions: string[]`（[schema](../backend/app/schemas/portal_config.py#L105)、[默认值](../backend/app/config/portal_config.py#L265)），后台「问答配置」页已可编辑（多行文本，一行一条，见 [AdminPage.tsx:608](../frontend/src/pages/AdminPage.tsx#L608)），并已通过公共配置 `/knowledge/config` 暴露给前端（[HomePage.tsx:494](../frontend/src/pages/HomePage.tsx#L494) 已算出 `qaHotQuestions`，但当前未用于「热门搜索」区块）。
- 搜索页筛选栏当前有 4 个下拉 + 排序（[SearchPage.tsx 筛选区](../frontend/src/pages/SearchPage.tsx)）：
  1. 知识库级别（`space_level`）
  2. 业务域（`domain` → 该业务域绑定的 `spaceIds`）
  3. 文档类型（`file_ext`）
  4. 标签（`tag`）
- 检索范围已按登录态切换（已落地）：未登录=后台启用库；已登录=后台启用库 ∪ 个人可见库。文件检索与搜索助手共用该范围。

---

## 2. 目标与范围

### 2.1 目标
1. 首页「热门搜索」改为展示后台配置的**热门问题**，点击直接进入搜索页并把问题填入搜索框、执行检索。
2. 搜索页二级筛选由「业务域」改为「知识空间」，按一级筛选（级别）+ 登录态动态展示可选空间。
3. 任一筛选条件变化后，**文件检索结果与搜索助手总结**都按新条件重新返回。

### 2.2 明确不做（范围外）
- **不新增后台侧边栏页**。热门问题继续在现有「问答配置」页维护（复用 `qa.hot_questions`）。
- **不改 `qa.hot_questions` 数据结构**，仍为有序 `string[]`。
- 不改 BiSheng 侧任何代码/接口。
- 首页「业务域导航卡片」、后台「业务域管理」均**不变**；本次仅替换**搜索页筛选栏**里的业务域下拉。

## 3. 名词

| 术语 | 含义 |
|---|---|
| 级别（space_level） | 公共 / 部门 / 团队 / 个人 四个层级 |
| 知识空间 | 一个具体的知识库（有 id、name、所属级别 space_level） |
| 当前范围 | 未登录=后台启用库；已登录=后台启用库 ∪ 个人可见库 |

## 4. 需求详述与验收标准

### R1 热门问题配置（复用现有，无侧边栏新增）
- 继续在后台「问答配置」页编辑 `qa.hot_questions`（多行文本，一行一条，保存后即生效）。
- **本需求无新增开发**，仅作为 R2/R3 的数据来源说明。
- 验收：在「问答配置」里增删热门问题并保存后，首页热门搜索内容随之变化（见 R2）。

### R2 首页「热门搜索」改读热门问题
- 「热门搜索」下拉列表的数据源由 `tags`（`rankedHotTags`）改为 `qa.hot_questions`（`qaHotQuestions`）。
- 取数：`config.qa.hot_questions`，去空白、过滤空串；**配置多少展示多少**，不复用 `hotTagsCount` 等上限，不做截断。
- **UI 兼容**：条目可能较多/较长，下拉区域需支持自动换行（或在超出时滚动），避免溢出或挤压布局；具体样式实现时处理。
- 列表为空时展示「暂无热门问题」（替换原「暂无热门标签」文案）。
- 验收：未登录/已登录均展示后台配置的**全部**热门问题；条目较多时布局正常；配置为空时显示空态文案。

### R3 点击热门问题进入搜索页并填充输入框
- 点击某条热门问题：关闭下拉 → 跳转到搜索页，将该问题文本作为**关键词**填入搜索框并触发检索（即 `/search?q=<问题>` 等价于用户在搜索框输入该问题并搜索）。
- 与现状区别：当前点击热门**标签**走的是按标签过滤（`buildTagSearchPath`）；热门**问题**应走**关键词检索**路径（等价于首页 `handleSearch` 用该问题作为 query）。
- 验收：点击任一热门问题后，搜索页搜索框显示该问题文本，结果区与搜索助手均按该关键词返回。

### R4 搜索页一级筛选（级别）保持不变
- 一级筛选固定为 5 项：`全部知识库`（空值）/ 公共空间 / 部门空间 / 团队空间 / 个人空间，**静态、不随权限裁剪**。
- ⚠️ 这与已上线的「一级筛选按权限动态裁剪级别」的改动冲突，需**回退该行为**（见 §7）。
- 验收：任意登录态下，一级筛选始终展示全部 5 项。

### R5 搜索页二级筛选改为「知识空间」（替换业务域）
- 二级筛选项从「业务域」替换为「知识空间」；业务域下拉**从搜索页筛选栏移除**。
- 选项 = 当前范围内、且符合一级筛选所选级别的知识空间；**列表随登录态变化**：
  - 未登录：来自 `config.spaces` 中 `enabled` 的空间（含 `space_level`）。
  - 已登录：在上面基础上**并入** `/knowledge/spaces`（登录态接口，按个人 token 返回）里的可见空间，按空间 id 去重。
- 与一级筛选联动：
  - 一级=`全部知识库`（空）：二级列出当前范围内**所有级别**的空间，默认项「全部空间」。
  - 一级=某级别：二级仅列出该级别的空间，默认项「全部」（= 该级别全部空间）。
  - **一级筛选变化时，二级筛选重置为默认项**（避免残留不属于新级别的空间）。
- 验收：
  - 未登录时二级仅出现后台启用库；登录后多出该用户个人可见的空间。
  - 切换级别后，二级选项与之匹配，且选择被重置。

### R6 筛选变化后联动刷新检索与搜索助手
- 任一筛选（级别 / 知识空间 / 文档类型 / 标签 / 排序）变化后：
  - **文件检索结果**按新条件重新请求；
  - **搜索助手总结**也按新条件（关键词 + 级别 + 所选空间）重新生成。
- 实现要点：搜索助手的取数范围必须与文件列表一致（已有规则），其副作用依赖需覆盖一级/二级筛选所对应的参数（`space_level`、所选 `spaceIds`、关键词）。
- 验收：改任一筛选后，结果列表与搜索助手内容同时更新，且二者范围一致。

### R7 按知识空间浏览入口（空关键词跳转）
- 原首页「业务域卡片」等以 `?domain=X` 进入搜索页的入口，改为：**跳转到搜索页，搜索框留空**，并**预选对应的知识空间**（二级筛选定位到该业务域绑定的知识空间），展示该空间下的内容（浏览模式）。
- 即：不带关键词、仅带空间范围进入搜索页；列表 = 该空间内文件，搜索助手按 R8 对该空间内容做总结。
- 一个业务域绑定一个知识空间（后台约定），故定位到单个空间；若存在多个绑定空间，取其范围作为空间过滤。
- 验收：点业务域卡片 → 搜索页搜索框为空、二级筛选显示对应空间、列表展示该空间内容。

### R8 搜索助手支持「无用户问题」时总结上下文
- 浏览模式（无关键词，仅有空间/级别范围）下，搜索助手仍要工作：对**当前范围内检索到的前 N 个文件摘要**做整体概述并返回，而非因「无问题」而空转或提示未找到。
- 需**修改搜索助手提示词**，区分两种情况：
  - 有用户问题：依据前 N 文件摘要，针对用户搜索意图归纳总结（现状）。
  - 无用户问题：对前 N 文件摘要做整体内容概述/要点提炼。
- 前端：搜索助手副作用的触发条件需放宽——当无关键词但存在空间/级别过滤（即处于有效搜索/浏览上下文）时也要触发。
- 其余约束不变（限 350 字、不输出字数统计、来源取前 N 文件）。
- 验收：浏览某知识空间（搜索框为空）时，搜索助手给出该空间内容的概述 + 来源文件。

## 5. 筛选 → 后端参数映射（后端无需改动）

二级筛选的「具体空间选择」与一级筛选的级别，最终落到既有检索参数：

| 前端选择 | `space_level` | `spaceIds` |
|---|---|---|
| 一级=全部 + 二级=全部空间 | 空 | 空（后端解析为整个当前范围） |
| 一级=某级别 + 二级=全部 | 该级别 | 空（后端按级别 + 范围解析） |
| 二级=某具体空间 | 同一级（或空） | `[该空间 id]` |

- 这些参数同时传给 `searchFiles` 与 `streamChatCompletion`（`knowledgeSpaceIds` + `space_level`）。
- 后端 `resolve_requested_space_ids(requested, space_level, extra_space_ids)` 已支持：`requested ∩ (启用库[按级别] ∪ 个人可见库)`，所选具体空间会被交集保留；个人可见库即使不在 config 中也已并入。**因此后端无需改动**。

## 6. 改动点（实现指引）

> 前端为主；后端仅 R8 需微调搜索助手提示词。不动数据模型、不动 BiSheng。

0. **后端 [chat_proxy_service.py](../backend/app/services/chat_proxy_service.py)（R8）**
   - `_build_search_summary_prompt` 增加「无用户问题」分支：当 `query` 为空时，改为对前 N 文件摘要做整体概述，不再输出「用户搜索/未找到」式措辞；有 `query` 时维持现状。
   - 其余（限 350 字、禁止字数统计、前 N 来源 citations）不变。

1. **首页 [HomePage.tsx](../frontend/src/pages/HomePage.tsx)**
   - 「热门搜索」区块数据源：`rankedHotTags` → `qaHotQuestions`（已存在，见 [L494](../frontend/src/pages/HomePage.tsx#L494)）；展示全部、不截断；样式兼容多条目。
   - 点击项：由 `navigate(buildTagSearchPath(tagName))` 改为按关键词跳转检索（等价于以该问题为 `q` 执行 `handleSearch`）。
   - 空态文案改为「暂无热门问题」。
   - 业务域卡片入口（R7）：改为跳搜索页、空关键词 + 预选对应知识空间（带该业务域绑定空间的 `spaceIds`，不带 `q`）。
2. **搜索页 [SearchPage.tsx](../frontend/src/pages/SearchPage.tsx)**
   - 一级筛选回退为静态 5 项（移除按权限裁剪 `availableSpaceLevels` 的逻辑）。
   - 二级筛选由「业务域」替换为「知识空间」：
     - 复用 `useAuth` + `fetchKnowledgeSpaces`（这两者是上一改动引入的，本次从「驱动级别」改为「驱动空间列表」）。
     - 由 `config.spaces`（启用库）+ 登录态下的 `/knowledge/spaces` 计算候选空间，按一级所选级别过滤、按 id 去重。
     - 选具体空间 → 以 `spaceIds=[id]` 进入检索；切级别时重置二级选择。
   - 移除 `domain` 下拉与 `createDomainFilterSearchParams` 在筛选栏的使用；R7 的入口改以 `spaceIds`（或预选空间参数）承载初始空间范围。
   - 搜索助手副作用触发条件放宽（R8）：无关键词但有空间/级别上下文时也触发；并确保文件检索与搜索助手副作用的依赖项覆盖新的二级筛选值，满足 R6。

上一轮已上线（master `0a60b5b`、测试环境已部署）的改动里，包含「搜索页**一级**筛选按登录权限动态裁剪可选级别」。本 PRD 的 R4 要求一级筛选恢复静态 5 项，故需：

- **复用**：原引入的 `useAuth` 与 `fetchKnowledgeSpaces` 登录态取数逻辑，**移用于 R5 的二级「知识空间」列表**。

其余已上线规则（检索范围按登录态切换、搜索助手基于前 10 文件摘要总结并限 350 字）**保持不变**。

## 8. 边界与异常

- **未配置热门问题**：首页热门搜索显示空态文案；不影响搜索框正常使用。
- **二级空间为空**（某级别在当前范围内无空间）：二级仅有「全部」默认项，或按实现展示空态；选择该级别检索结果可能为空，属预期。
- **登录态变化**：登录/登出后，二级空间列表需相应刷新（`useAuth` 的 `user` 变化驱动重新取数）。
- **`/knowledge/spaces` 失败或 401**：降级为仅用启用库（`config.spaces`）渲染二级，不阻断页面。
- **未登录用户选择个人级别**：未登录范围内通常没有个人空间 → 二级为空，结果为空，属预期。

## 9. 决策记录（原开放问题，已确认）

1. **业务域入口去向** → 已定：仍跳搜索页，但**搜索框留空、预选对应知识空间**进入浏览模式（见 R7）。不再使用 `?domain=X` 的标签/域过滤语义。
2. **首页热门问题展示条数** → 已定：**不复用** `hotTagsCount`，配置多少展示多少、不截断；UI 需兼容较多条目（见 R2）。
3. **二级具体空间与级别的回填** → 已定：**不回填**级别；二级选具体空间时按 `spaceIds=[id]` 检索，`space_level` 不强制带。
